# ナレッジベース：SSBJ対応自律型AIエージェント開発ガイド

## 1. プロジェクト概要

本プロジェクトは、2026年時点のSSBJ（特にS2：気候関連開示）に対応するため、単なる文章生成を超え、データ収集・計算・戦略評価・監査シミュレーションまでを一貫して行う**マルチエージェント型AIシステム**を構築するものである。

---

## 2. システムアーキテクチャ

システムは、各専門領域に特化したエージェントが協調する**「MAS（マルチエージェント・システム）」**を採用する。

### 2.1 構成要素

|**要素**|**役割**|**技術スタック（推奨）**|
|---|---|---|
|**脳 (Reasoning)**|基準解釈・意思決定|GPT-5 / Claude 3.5クラス|
|**計画 (Planning)**|タスク分解・自己修正|LangGraph (Stateful Workflow)|
|**記憶 (Memory)**|文脈維持・基準参照|GraphRAG (Neo4j + VectorDB)|
|**ツール (Tools)**|PDF解析・計算・証跡管理|Python (PyMuPDF/Pandas), API Connectors|

## 3. シナリオ分析とデータ戦略

SSBJ S2で最難関となる「シナリオ分析（1.5℃/4.0℃）」において、AIが扱うべきデータ構造を定義する。

### 3.1 ミニマムデータリスト（入力データ）

- **内部データ:** 拠点座標、Scope 1/2排出量、エネルギーコスト、固定資産簿価、財務利益率。
    
- **外部データ:** 炭素税予測（IEA等）、物理的リスク確率（IPCC等）、為替レート予測。
    

### 3.2 財務影響試算の論理（Python実行）

LLMによる直接計算ではなく、以下の数式をコード実行させる。

$$\text{Total Impact} = (\text{Emissions} \times \Delta\text{Carbon Price}) + (\text{Asset Value} \times \text{Damage Prob.})$$

## 4. 意思決定支援と高度な分析機能

### 4.1 新旧対照分析 (Diff Analysis)
有価証券報告書のPDFを読み込み、SSBJ基準とのギャップを特定した上で、具体的な修正案を生成する。

### 4.2 財務影響シミュレーション
AIは単一のドラフトではなく、「経営判断の選択肢」を提示する。
炭素価格やシナリオ環境を動的に変更し、将来の財務諸表（損益計算書）へのインパクトを可視化する。

- **SSBJ準拠度:** 基準要求事項を網羅しているか。
    
- **投資家訴求力:** 戦略的レジリエンスが示されているか。
    
- **監査コスト:** 証跡の複雑さと検証容易性。
    

### 4.2 解説付きドラフト（Explainable AI）

ドラフトの各パラグラフに対し、以下の「コメンタリー（解説）」を付帯させる。

- **適用基準:** SSBJ S2 第XX項に基づく記述である旨。
    
- **意図:** なぜこの数値を強調し、この不確実性を明記したかの論理。

## 5. カスタム監査モデルとプレ・アシュアランス

監査法人からの指摘を先回りして解消するための防御的機能。

### 5.1 過去の指摘ナレッジベース（JSONL形式）

過去のQ&A記録から機密情報を除き、AIが学習可能な形式に変換する。

JSON

```
{
  "metadata": {"category": "Transition Risk", "severity": "High", "clause_id": "S2-34"},
  "audit_finding": "炭素価格の参照元が不明瞭であり、試算の客観性に欠ける。",
  "management_response": "IEA NZE 2024年版の価格レンジに基づき、感度分析を追記した。"
}
```

### 5.2 プレ・アシュアランス・エージェントの挙動

1. **Red-Teaming:** ドラフトに対し「監査人ペルソナ」で論理的弱点を指摘。
    
2. **Evidence Binding:** 文中の全数値に対し、元ファイル（Excel等）のセル番地をメタデータとして紐付け。
    
3. **Traceability:** 計算プロセスをPythonログとして残し、監査人が検証可能な「証跡パッケージ」を自動生成。

## 6. 実装時のエンジニア向け工夫ポイント

- **ハルシネーション防止:** 出力された数値をソースデータと再照合する「セマンティック・チェッカー」の実装。
    
- **動的ルーティング:** 立地条件（沿岸/内陸）に応じて、参照するリスクデータ（洪水/渇水）をAIが自動選択。
    
- **匿名化パイプライン:** 過去の知見を学習させる際、個人情報や機密数値を自動マスキングする仕組みの構築。
    

---

## 7. 導入ロードマップ

1. **Phase 1:** 過去の開示資料・監査Q&Aの構造化とRAG構築。
    
2. **Phase 2:** 特定拠点（1工場）を対象としたシナリオ分析PoC。
    
3. **Phase 3:** 複数シナリオ比較と「重要ポイント解説」機能の統合。
    
4. **Phase 4:** 監査対応用「証跡パッケージ」自動生成機能のリリース。
